## KPI-конструктор — финальное состояние

### Данные в БД
- `kpi_indicators` — **124** показателя (не 126 — исправлена ошибка), 3 общих `is_common`
- `kpi_criteria` — поля `sub_type`, `order` добавлены (миграция k4l5m6n7o8p9)
- `kpi_role_cards` — 91 карточка, `kpi_role_card_indicators` — 261 привязка
- `subordination` — таблица в PostgreSQL (миграция j3k4l5m6n7o8)

### 6 типов показателей
| Тип | Оценивает | Особенность |
|---|---|---|
| binary_auto | AI | По трудозатратам Redmine |
| binary_manual | Руководитель | ✅/❌ |
| multi_binary | Руководитель | N подпоказателей, все должны быть ✅ |
| threshold | Система | числитель/знаменатель → % → пороги |
| multi_threshold | Система | N числовых подпоказателей, все должны выполниться |
| quarterly_threshold | Система | отдельные пороги Q1/Q2/Q3/Q4 |

### Workflow новых показателей
```
Начальник управления / Зам. директора → создаёт (draft)
HR → проверяет (review)      ← UI ещё не реализован
Admin → утверждает (active)  ← UI ещё не реализован
```

### Admin-панель — что работает
- **Подчинённость:** Список + По подразделениям (верхний уровень), sticky-шапка, бейджи карточек, сохранение в БД
- **KPI-карточки:** сайдбар по подразделениям, wizard создания, редактирование весов
- **KPI-показатели:** конструктор всех 6 типов, визуальный редактор порогов, валидация
- Хелпер `normalizeUnit()` → `frontend/src/utils/admin.ts`
