> Версия: 1.0 | Дата: 2026-05-02 | Статус: финал

## ПРИНЦИП: Redmine — источник структуры

Портал **не создаёт** должности и подразделения самостоятельно.  
Вся организационная структура приходит из Redmine через «Импорт из People».

```
Redmine                          KPI-портал
──────────────────────           ─────────────────────────────
Пользователи (81 чел.)    →      users (таблица)
Должности (position)      →      привязка к kpi_role_cards
Подразделения             →      группировка карточек
Иерархия (руководители)   →      subordination (evaluator)
```

**Исключение:** KPI-карточки и веса показателей создаются только в портале.

---

## ЖИЗНЕННЫЙ ЦИКЛ ПРИ ИЗМЕНЕНИИ СТРУКТУРЫ

```
1. HR добавляет нового сотрудника / должность в Redmine
        ↓
2. Admin → «Импорт из People» в портале
        ↓
3. Система показывает diff:
   «Найдено N должностей без KPI-карточки: [список]»
        ↓
4. Admin переходит в таб «KPI-карточки»
   → видит новые должности с бейджем ⚠️
        ↓
5. Для каждой новой должности:
   a) Создать новую карточку (пустую)  → добавить показатели вручную
   b) Скопировать с похожей должности  → отредактировать отличия
        ↓
6. Назначить руководителя в табе «Подчинённость» → кнопка «Изменить»
        ↓
7. Готово — при следующем периоде сотрудник получит карточку
```

## ПРАВИЛА KPI-КАРТОЧЕК

### Создание
- Карточка создаётся вручную admin'ом или HR
- Источник: новая пустая ИЛИ копия существующей карточки
- После создания — вступает в силу со **следующего** отчётного периода

### Редактирование
- Изменение весов и состава показателей — только через admin-панель
- Изменения вступают в силу со **следующего** периода (не текущего)
- `is_common` показатели (исполнительская дисциплина, трудовой распорядок, охрана труда) — меняет только HR/admin, они есть у всех 91 сотрудника

### Версионирование
- При сохранении изменений карточки — создаётся новая версия (`v1 → v2`)
- Старая версия остаётся для исторических отчётов
- В интерфейсе показывается актуальная версия

### Копирование
- При создании через «Скопировать с...» — копируются показатели и веса
- Общие (`is_common`) показатели добавляются автоматически и не копируются
- После копирования карточку нужно отредактировать под конкретную должность

## ПРАВИЛА ПОДЧИНЁННОСТИ

### Источник данных
- Иерархия (кто кому подчиняется) — из Redmine через `subordination.json` / таблицу `subordination`
- Руководитель для review — поле `evaluator` в subordination

### Кто оценивает KPI
```
Сотрудник → его evaluator (прямой руководитель) проверяет binary_manual
Evaluator = null → подчиняется директору, вне системы review
```

### Рекурсивная подчинённость
- Руководитель видит всех подчинённых по цепочке (не только прямых)
- Пример: Зам. директора → Начальник управления → все сотрудники управления
- Реализовано через `_get_effective_subordinate_ids` в `review.py`

### Ограничения
- Нельзя создать цикл (A → B → A)
- У каждого сотрудника только один прямой руководитель (evaluator)
- Директор — вне системы (evaluator = null), его подчинённые оцениваются замами

## UI ADMIN-ПАНЕЛИ: ТАБ «ПОДЧИНЁННОСТЬ»

### Два режима отображения
| Режим | Назначение |
|---|---|
| Список | Быстрый поиск конкретной должности, изменение руководителя |
| Дерево | Обзор иерархии, проверка правильности структуры, статус карточек |

### Статус карточки в дереве
- `✅ карточка` — есть запись в `kpi_role_cards` для этой должности
- `⚠️ нет карточки` — должность есть в `users`, карточки нет → клик ведёт в KPI-карточки

### Действия
- **Изменить** (в списке и дереве) — сменить руководителя (evaluator)
- **Обновить** — перечитать из БД без импорта из Redmine
- **Импорт из People** — синхронизировать с Redmine, после чего показать diff

## UI ADMIN-ПАНЕЛИ: ТАБ «KPI-КАРТОЧКИ»

### Структура страницы
```
[Левый сайдбар]              [Правая часть]
────────────────             ────────────────────────────────
Все карточки  91             [+ Создать карточку]
─────────────────
Руководство    4             ┌──────────────────────────────┐
Уп. орг. обесп 12            │ Название должности  pos_id   │
ЕАС           13             │ подразделение · версия       │
КЗА            8             │                  [Открыть →] │
...                          └──────────────────────────────┘
─────────────────
⚠️ Без карточки 2            ┌──────────────────────────────┐
                             │ ⚠️ Новая должность           │
                             │             [Создать карт. →]│
                             └──────────────────────────────┘
```

### Wizard создания карточки
- **Шаг 1:** Название, подразделение, код (авто или ручной)
- **Шаг 2:** Пустая карточка ИЛИ скопировать с существующей
- После создания — сразу открывается в режиме редактирования
