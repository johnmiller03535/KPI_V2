# ЗАДАЧА: Хотфикс — группа не сохраняется при создании показателя

> Файл для Claude Code | Дата: 2026-05-03 | Приоритет: 🔴 Критический

---

## Проблема

При создании показателя через «+ Добавить показатель» выбранная группа
не сохраняется в БД. Показатель создаётся без группы (или с дефолтным значением).

При редактировании существующего показателя группа сохраняется корректно.

---

## Диагностика (выполнить в браузере)

Открыть DevTools → Network → XHR/Fetch.
Нажать «+ Создать показатель» с выбранной группой.
Найти POST-запрос к `/api/admin/indicators/`.
Посмотреть **тело запроса (Request Payload)**:

```json
// Что должно быть:
{
  "name": "Тестовый пороговый показатель",
  "group_name": "Закупочная деятельность",   ← это поле должно быть здесь
  "formula_type": "threshold",
  ...
}

// Что скорее всего есть:
{
  "name": "Тестовый пороговый показатель",
  "group_name": "",    ← пустая строка или null
  ...
}
```

---

## Вероятная причина

Форма использует два разных state-ключа для группы при создании vs редактировании.
Например:
- При редактировании: `formData.group_name` → отправляется в PATCH
- При создании: `newIndicator.group` → отправляется в POST, но поле в API называется `group_name`

**Или:** onChange дропдауна обновляет `group` а handleSubmit читает `group_name`.

---

## Исправление

Найти в `page.tsx` функцию `handleCreateIndicator` (или аналогичную).
Убедиться что поле группы читается из того же state-ключа что обновляет onChange:

```typescript
// В onChange дропдауна группы:
setNewIndicatorData(prev => ({ ...prev, group_name: e.target.value }))
//                                       ^^^^^^^^^^^

// В handleSubmit:
const body = {
  name: newIndicatorData.name,
  group_name: newIndicatorData.group_name,   // тот же ключ!
  formula_type: newIndicatorData.formula_type,
  ...
}
```

Также проверить что поле `group_name` присутствует в Pydantic-схеме
`POST /api/admin/indicators/` на бэкенде и маппится корректно.
