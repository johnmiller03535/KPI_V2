## БАГ 1 — Шапка модалок скрыта за навбаром (оба типа: редактирование и просмотр)

### Проблема
Предыдущий фикс (`padding-top: 72px`) не сработал — шапка всё ещё скрыта.

### Диагностика
Проверить в DevTools → Elements что реально применяется к overlay-div.
Возможно `align-items: center` на overlay переопределяет `padding-top`.

### Решение — другой подход
Вместо padding на overlay — добавить `margin-top` на саму модалку:

```tsx
{/* Overlay — простой fixed контейнер без flex-centering */}
<div style={{
  position: 'fixed',
  inset: 0,
  overflowY: 'auto',       // скролл на overlay
  zIndex: 100,
  background: 'rgba(0,0,0,0.75)',
  padding: '80px 24px 24px', // 80px сверху = навбар(64) + отступ(16)
}}>
  {/* Модалка — просто block, не flex-item */}
  <div style={{
    width: '700px',
    maxWidth: '95vw',
    margin: '0 auto',       // центрирование по горизонтали
    background: 'var(--card)',
    borderRadius: '8px',
    // НЕТ position, НЕТ transform, НЕТ top/left
  }}>
    {/* содержимое */}
  </div>
</div>
```

Ключевое: скролл на overlay, а не на модалке. Модалка просто block внутри.
Это проще и надёжнее чем flex + padding-top.

Применить к **обеим** модалкам: редактирование (IndicatorFormModal) и просмотр (view modal).

## ЗАДАЧА 2 — Редизайн карточки просмотра показателя

### Проблема
Вся информация слита в один блок — трудно читать. Нужно визуально разделить секции.

### Текущий вид (плохо)
```
Критерий оценки
  [текст критерия]
  [методика курсивом]
  Числитель: ...
  Поле ввода: ...
  ПОРОГИ
  >=67% → 100 баллов
  <67% → 0 баллов
```

### Желаемый вид (хорошо)
```
┌─────────────────────────────────────────────┐
│  Название показателя                         │
│  [Абсолютный] [active] [Правовое обесп.]    │
├─────────────────────────────────────────────┤
│  КРИТЕРИЙ ОЦЕНКИ                            │
│  Обеспечение качественного согласования...  │
│                                             │
│  💡 Методика: Отсутствие более 2-х...       │  ← курсив, приглушённый
├─────────────────────────────────────────────┤
│  ЧИСЛОВОЙ ПОКАЗАТЕЛЬ                        │  ← только для threshold-типов
│  Поле ввода: Количество повторных...        │
│  Нарастающим итогом: нет                    │
├─────────────────────────────────────────────┤
│  ПРАВИЛА ОЦЕНКИ                             │
│  ≤ 2  →  100 баллов                        │  ← каждое правило на строке
│  > 2  →    0 баллов                        │
├─────────────────────────────────────────────┤
│  Используется в 1 карточках  [Редактировать]│
└─────────────────────────────────────────────┘
```

### Требования к layout
- Каждая секция отделена горизонтальным разделителем (`border-top: 1px solid rgba(0,229,255,0.15)`)
- Заголовки секций: `КРИТЕРИЙ ОЦЕНКИ`, `ЧИСЛОВОЙ ПОКАЗАТЕЛЬ`, `ПРАВИЛА ОЦЕНКИ` — маленькие капслок, цвет `var(--accent)`, font-size 11px
- Методика — курсив, цвет `var(--muted)`, с левой полоской `border-left: 2px solid var(--accent)`
- Правила оценки — каждое правило в отдельной строке, оператор + значение выровнены, балл справа
- Секция «ЧИСЛОВОЙ ПОКАЗАТЕЛЬ» — показывать только для `threshold`, `multi_threshold`, `quarterly_threshold`, `absolute_threshold`
- Для `binary_*` — не показывать секцию числового показателя

### Правила оценки — форматирование
```typescript
// Для threshold/multi/quarterly — с %:
`${rule.operator} ${rule.value}%  →  ${rule.score} баллов`

// Для absolute_threshold — без %:
`${rule.operator} ${rule.value}  →  ${rule.score} баллов`

// Для quarterly — сгруппировать по кварталам:
Q1:  >= 15%  →  100 баллов
     <  15%  →    0 баллов
Q2:  >= 25%  →  100 баллов
     ...
```
