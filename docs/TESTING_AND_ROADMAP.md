## Реализовано

### Инфраструктура и данные
- Синхронизация сотрудников из Redmine (81 чел., 9 подразделений)
- Импорт матрицы подчинённости из People export (Excel → `subordination.json`)
- `GET /api/admin/data-health` — проверка состояния данных (сотрудники без должности, без Telegram, без подчинённого)
- Таб «Подчинённость» в admin-панели — просмотр дерева иерархии

### Периоды и задачи
- CRUD периодов (draft → active → review → closed)
- Создание KPI-задач в Redmine (dry_run поддержан)
- Исключения сотрудников из периода (dismissed/transferred/maternity)

### KPI-форма сотрудника
- Новый флоу AI-оценки: `load-summary` → редактирование текста → `generate-summary` → `submit`
- Трудозатраты Redmine обогащаются subject задачи (separate API calls + asyncio.gather)
- Классификация: рабочие / отпускные записи (по project.name и activity.name)
- AI-саммари из поля `comments` как основного источника данных

### Проверка руководителем
- `manager_override` для binary_auto показателей (переопределяет AI-оценку)
- binary_manual показатели: кнопки ✅/❌ без перезагрузки страницы
- Стандартные формулировки для `is_common` показателей (из `COMMON_KPI_TEXTS`)
- Блокировка «Утвердить» при наличии непроставленных ручных оценок

### PDF-отчёт
- Период в формате `дд.мм.гггг — дд.мм.гггг` (из `period.date_start/date_end`)
- Подпись: должность руководителя из KPI_Mapping + ФИО из таблицы employees
- Колонка «Мероприятия»: `summary_text` сотрудника (не AI reasoning)
- Стандартные формулировки для общих показателей в основной таблице
- Удалена страница «Детализация показателей»

### Дизайн и UX
- Dark Cyber дизайн-система (Orbitron, Exo 2, glassmorphism)
- `computeScore` учитывает `manager_override` везде (dashboard, review, detail)
- `src/lib/kpiScore.ts` — единый источник расчёта оценки
