## ТИПЫ KPI ПОКАЗАТЕЛЕЙ (все 8 реализованы ✅)

| Тип | Кто оценивает | Как |
|---|---|---|
| binary_auto | AI → руководитель (финал) | AI саммари из Redmine → подсказка руководителю, он ставит ✅/❌ |
| binary_manual | Руководитель | Кнопки ✅/❌ |
| multi_binary | Руководитель | Каждый подпоказатель ✅/❌, все нужны; у каждого своя методика + тексты |
| threshold | Сотрудник | Числитель/знаменатель → % → пороги |
| multi_threshold | Сотрудник | Несколько пар числитель/знаменатель; у каждого своя методика |
| multi_mixed 🆕 | Сотрудник (threshold) + руководитель (binary_manual) | Смешанные подпоказатели; все нужны |
| quarterly_threshold | Сотрудник | % → пороги по кварталам; годовой = Q4 |
| absolute_threshold | Сотрудник | Одно число → квартальные пороги; числитель/знаменатель |

### Логика multi_mixed (sub_indicators JSONB)
```json
// threshold-подпоказатель:
{"id":"uuid","name":"...","sub_type":"threshold","formula_desc":"...","numerator_label":"...","denominator_label":"...","cumulative":false,"rules":[...]}
// binary_manual-подпоказатель:
{"id":"uuid","name":"...","sub_type":"binary_manual","formula_desc":"...","positive_text":"...","negative_text":"..."}
```

## ФЛОУ AI-ОЦЕНКИ (финальный)

```
1. Сотрудник → «Загрузить саммари»
   → time_entries из Redmine (поле comments = основа)
   → для каждого binary_auto показателя отдельное саммари через YandexGPT

2. Сотрудник редактирует саммари (textarea под каждым показателем)

3. «Отправить на проверку»
   → AI оценивает каждый binary_auto по его саммари
   → статус submitted, форма read-only
   → уведомление руководителю в Telegram

4. Руководитель → /review
   → видит саммари + AI-оценку + уверенность
   → может переопределить AI (manager_override)
   → оценивает binary_manual (✅/❌)
   → утверждает

5. PDF генерируется и прикрепляется к задаче в Redmine
```
