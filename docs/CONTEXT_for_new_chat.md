## ФЛОУ AI-ОЦЕНКИ (финальный)

```
1. Сотрудник → «Загрузить саммари»
   → time_entries из Redmine (поле comments = основа)
   → для каждого binary_auto показателя отдельное саммари через YandexGPT

2. Сотрудник редактирует саммари (textarea под каждым показателем)

3. «Отправить на проверку»
   → AI оценивает каждый binary_auto по его саммари
   → статус submitted, форма read-only
   → уведомление руководителю в Telegram

4. Руководитель → /review
   → видит саммари + AI-оценку + уверенность
   → может переопределить AI (manager_override)
   → оценивает binary_manual (✅/❌)
   → утверждает

5. PDF генерируется и прикрепляется к задаче в Redmine
```

## ТИПЫ KPI ПОКАЗАТЕЛЕЙ (все 7 реализованы в конструкторе ✅)

| Тип | Кто оценивает | Как |
|---|---|---|
| binary_auto | AI (YandexGPT) | По саммари трудозатрат |
| binary_manual | Руководитель | Кнопки ✅/❌ |
| multi_binary | Руководитель | Каждый подпоказатель отдельно ✅/❌, нужны все |
| threshold | Сотрудник вводит факт | Числитель/знаменатель → % → пороги |
| multi_threshold | Сотрудник (несколько пар) | Все подпоказатели должны быть выполнены |
| quarterly_threshold | Сотрудник | Пороги меняются по кварталам |
| absolute_threshold | Сотрудник | Одно число → прямое сравнение с порогами |
