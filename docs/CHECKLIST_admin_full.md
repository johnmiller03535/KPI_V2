## 🔵 ТАБ «ПОДЧИНЁННОСТЬ»

### Режим «Список»
- [ ] 91 должность отображается
- [ ] Заголовок `ДОЛЖНОСТЬ / ПОДРАЗДЕЛЕНИЕ / РУКОВОДИТЕЛЬ / КАРТОЧКА` виден при скролле до конца
- [ ] Все 91 должность имеют бейдж `✅ карточка` (зелёный)
- [ ] Кнопка «Изменить» открывает модалку смены руководителя
- [ ] После сохранения — нет ошибки, руководитель обновился
- [ ] После перезагрузки страницы — изменение сохранилось (пишется в PostgreSQL)

### Режим «По подразделениям»
- [ ] Заголовок «МАТРИЦА ПОДЧИНЕНИЯ» + кнопки видны при скролле
- [ ] Левая панель: ~9 управлений (верхний уровень), без дублей отделов
- [ ] «Руководство» первым в списке
- [ ] Счётчики рядом с управлениями корректны (сумма = 91)
- [ ] Клик на управление → правая часть показывает только его должности
- [ ] «⚠️ Без карточки» внизу, показывает 0
- [ ] Кнопка «Изменить» работает в этом режиме

### Импорт
- [ ] «Импорт из People» завершается без ошибки
- [ ] После импорта — баннер «Найдено N должностей без карточки» показывается если есть, не показывается если нет

## 🔵 ТАБ «KPI-КАРТОЧКИ»

### Общий вид
- [ ] Заголовок «КАРТОЧКИ ДОЛЖНОСТЕЙ» + кнопка «+ Создать карточку» видны при скролле
- [ ] Левый сайдбар: верхний уровень (~9 управлений), без дублей
- [ ] «Руководство» первым
- [ ] «⚠️ Без карточки» внизу, показывает 0
- [ ] «Все карточки» = 91

### Просмотр карточки
- [ ] Клик на подразделение → список карточек обновился
- [ ] Кнопка «Открыть →» показывает карточку с показателями
- [ ] Карточка: название, pos_id, версия, статус, подразделение
- [ ] Список показателей: название, тип, «Общий» бейдж, вес в %
- [ ] Сумма весов = 100% с зелёным «✓ Верно»
- [ ] Кнопка «← Назад к списку» возвращает в список

### Редактирование карточки
- [ ] Кнопка «Редактировать» переводит в режим редактирования
- [ ] Веса редактируются (поля ввода)
- [ ] Стрелки ▲▼ меняют порядок показателей
- [ ] Корзина удаляет показатель из карточки
- [ ] «+ Добавить показатель» открывает выбор из библиотеки
- [ ] Сумма весов пересчитывается в реальном времени
- [ ] «Сохранить» сохраняет без ошибки
- [ ] «Отмена» откатывает изменения

### Wizard «+ Создать карточку»
- [ ] Шаг 1: поля «Название», «POS_ID», «Подразделение» заполняются
- [ ] «Далее →» неактивна при пустых обязательных полях
- [ ] Шаг 2: радиокнопки «Пустая» / «Скопировать с...»
- [ ] При выборе «Скопировать» — дропдаун карточек появляется
- [ ] «Создать карточку» создаёт и сразу открывает редактирование
- [ ] При копировании — показатели скопированы с источника
- [ ] «Отмена» / ESC закрывает без сохранения

## 🔵 ТАБ «KPI-ПОКАЗАТЕЛИ»

### Общий вид
- [ ] Строка поиска + кнопка «+ Добавить показатель» видны при скролле
- [ ] Заголовок таблицы `НАЗВАНИЕ / ТИП / ИСПОЛЬЗУЕТСЯ` виден при скролле до конца
- [ ] Счётчик «Все» = **126**
- [ ] Левый сайдбар: все группы с корректными счётчиками, сумма = 126
- [ ] «Общие показатели» = 3

### Фильтрация
- [ ] Клик на группу → список фильтруется по группе
- [ ] Поиск по названию работает
- [ ] Поиск + фильтр по группе работают вместе

### Просмотр показателя
- [ ] Кнопка «Просмотр» открывает модалку
- [ ] Модалка: название, тип, группа, критерий, пороги, «используется в N карточках»
- [ ] Закрывается на ✕ и клик вне модалки

### Редактирование показателя
- [ ] Кнопка «Редактировать» открывает форму
- [ ] Для `binary_auto/manual`: поля название, группа, критерий
- [ ] Для `threshold`: дополнительно числитель, знаменатель, пороги JSON
- [ ] «Сохранить» сохраняет без ошибки
- [ ] После сохранения — изменения видны в списке

### Создание показателя
- [ ] «+ Добавить показатель» открывает форму
- [ ] Select типа в начале формы (все 5 типов в списке)
- [ ] При смене типа — форма адаптируется (threshold показывает доп. поля)
- [ ] Обязательные поля валидируются
- [ ] После создания — новый показатель появляется в списке
- [ ] Счётчик увеличивается на 1

## 🔵 SMOKE-TEST: СКВОЗНЫЕ СЦЕНАРИИ

### Сценарий А — Новая должность без карточки
- [ ] Открыть «Подчинённость» → «По подразделениям»
- [ ] Найти любую должность с `⚠️ нет карточки` (если есть)
- [ ] Кликнуть на бейдж → перешли в «KPI-карточки»
- [ ] Создать карточку через wizard → карточка создана

### Сценарий Б — Изменить руководителя
- [ ] «Подчинённость» → «Изменить» для любой должности
- [ ] Сменить руководителя → сохранить
- [ ] Перезагрузить страницу → изменение сохранилось
- [ ] Вернуть обратно

### Сценарий В — Добавить показатель в карточку
- [ ] «KPI-показатели» → «+ Добавить показатель» → создать тестовый binary_auto
- [ ] «KPI-карточки» → открыть любую карточку → «Редактировать»
- [ ] «+ Добавить показатель» → найти созданный → добавить с весом
- [ ] Проверить сумму весов → «Сохранить»
