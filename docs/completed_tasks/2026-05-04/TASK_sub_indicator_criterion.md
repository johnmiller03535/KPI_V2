# ЗАДАЧА: Добавить поле «критерий» для каждого подпоказателя multi_binary

> Файл для Claude Code | Дата: 2026-05-04 | Приоритет: 🔴

---

## Проблема

Для `multi_binary` каждый подпоказатель должен иметь:
1. `description` — краткое название подпоказателя (уже есть)
2. `criterion` — детальный критерий оценки для этого подпоказателя (отсутствует)

Сейчас `kpi_criteria.description` используется как описание, но поля критерия нет.

---

## ЧАСТЬ 1 — Миграция БД

```sql
-- Добавить поле criterion в kpi_criteria
ALTER TABLE kpi_criteria ADD COLUMN IF NOT EXISTS sub_criterion TEXT DEFAULT NULL;
-- sub_criterion — критерий оценки конкретного подпоказателя
-- Используется только для sub_type = 'sub_binary'
```

---

## ЧАСТЬ 2 — Backend

### Схемы (`backend/app/schemas/admin.py`)
Добавить `sub_criterion: str | None = None` в схемы критериев.

### API
В `GET /api/admin/indicators/` и `GET /api/admin/indicators/{id}` —
включать `sub_criterion` в ответ для записей с `sub_type = 'sub_binary'`.

В `POST` и `PATCH /api/admin/indicators/` —
принимать и сохранять `sub_criterion` для подпоказателей.

---

## ЧАСТЬ 3 — Frontend: форма редактирования

Для каждого подпоказателя `multi_binary` — два поля вместо одного:

```
ПОДПОКАЗАТЕЛИ * (минимум 2)

1.  Описание (кратко):
    [Отсутствие привлечения к административной ответственности    ] 🗑

    Критерий оценки (как проверять):
    [textarea: Проверить отсутствие штрафов и предписаний за период]

2.  Описание (кратко):
    [Отсутствие удовлетворённых исковых требований                ] 🗑

    Критерий оценки (как проверять):
    [textarea: Проверить отсутствие судебных решений о взыскании   ]

[+ Добавить подпоказатель]
```

Поле «Критерий» — необязательное (может быть пустым).
Подсказка под критерием: «Как именно руководитель проверяет этот подпоказатель»

---

## ЧАСТЬ 4 — Frontend: модалка просмотра

Для `multi_binary` показывать `sub_criterion` под каждым подпоказателем если заполнен:

```
ПОДПОКАЗАТЕЛИ
Руководитель оценивает каждый отдельно. Все должны быть ✅

1. Отсутствие привлечения к административной ответственности
   ↳ Проверить отсутствие штрафов и предписаний за период   ← курсив, приглушённый

2. Отсутствие удовлетворённых исковых требований
   ↳ Проверить отсутствие судебных решений о взыскании      ← курсив, приглушённый
```

Если `sub_criterion` пустой — показывать только `description` без строки `↳`.
