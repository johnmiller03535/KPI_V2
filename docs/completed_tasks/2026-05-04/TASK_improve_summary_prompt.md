# Задача: Улучшить промпт генерации саммари
> Файл: backend/app/services/kpi_engine_service.py

## Изменения в промпте generate_summary_from_tasks

### 1. Убрать период из саммари
Период не упоминать в тексте — он есть в отчёте отдельно.

### 2. Группировать повторяющиеся задачи с указанием количества

Перед передачей в промпт — агрегировать задачи:
```python
from collections import Counter

task_counts = Counter(e['issue']['subject'] for e in work_entries)

# Формировать список с количеством:
tasks_text = "\n".join(
    f"- {task} (×{count})" if count > 1 else f"- {task}"
    for task, count in task_counts.most_common()
)
```

### 3. Обновить промпт

```python
prompt = f"""Ты — система подготовки отчётов KPI государственного учреждения.
Составь формальное саммари выполненных работ сотрудника.

Должность: {role_name}
Показатель KPI: {indicator.indicator}

Выполненные задачи (×N = количество раз):
{tasks_text}

{absence_note}

Требования:
1. Деловой стиль. Напиши столько предложений сколько нужно, чтобы полно отразить все виды выполненных работ. Не сокращай если видов работ много.
2. Период НЕ упоминать
3. Повторяющиеся мероприятия указывать с количеством:
   "Проведено 4 совещания по...", "Подготовлено 3 аналитических отчёта..."
4. Группируй по смыслу: совещания, аналитика, разработка, координация
5. Только факты из списка — не придумывай
6. Если список пуст — "Рабочие задачи за период не зафиксированы"
"""
```
