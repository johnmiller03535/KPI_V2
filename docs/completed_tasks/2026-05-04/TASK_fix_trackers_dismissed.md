## БАГ 1 — Неправильный трекер при создании задач (КРИТИЧНО)

### Проблема
При создании задач в Redmine все сотрудники подразделения получают **один трекер** 
вместо индивидуального трекера по своему `role_id`.

Пример (ААД, 11 сотрудников):
- Все получили: `KPI_ОТЧЁТ_ААД_НАЧ_081`
- Должны были получить: `KPI_ОТЧЁТ_ААД_НАЧ_081`, `KPI_ОТЧЁТ_ААД_ЗАМ_082`, 
  `KPI_ОТЧЁТ_ААД_ЗАМ_НАЧ_ОТД_083`, `KPI_ОТЧЁТ_ААД_КОН_085` и т.д.

### Где искать баг
Файл: `backend/app/services/period_service.py` (или `task_service.py`)
Метод создания задач в Redmine — функция типа `create_redmine_tasks()`.

### Логика трекера (правильная)
```
для каждого сотрудника:
  role_id = KPI_Mapping.get_role_id(employee.position_id)
  tracker_name = f"KPI_ОТЧЁТ_{role_id}"  # например "KPI_ОТЧЁТ_ЦТР_КОН_075"
  
  # Найти tracker_id в Redmine по имени трекера
  tracker_id = redmine_trackers[tracker_name]
  
  # Создать задачу с этим tracker_id
  create_issue(tracker_id=tracker_id, assigned_to=employee.redmine_id, ...)
```

### Что проверить
1. Как сейчас определяется `tracker_id` при создании задачи — там наверняка берётся
   первый трекер подразделения или хардкод
2. Список трекеров из Redmine:
   ```bash
   curl -s "https://kkp.rm.mosreg.ru/trackers.json" \
     -H "X-Redmine-API-Key: 5f498fb9d36bd82c84edef1851e921b84a302dfa" \
     --insecure | python3 -m json.tool
   ```
3. Убедиться что имя трекера в Redmine точно совпадает с паттерном `KPI_ОТЧЁТ_{role_id}`

### После исправления
- Удалить все текущие задачи из Redmine (или они удалятся каскадно при удалении submissions)
- Удалить submissions через БД:
  ```sql
  DELETE FROM kpi_submissions;
  ```
- Пересоздать задачи через UI: /admin/periods → «Создать задачи»
- Проверить в Redmine что у каждого сотрудника свой трекер

## БАГ 2 — Нет просмотра уволенных сотрудников в admin-панели

### Проблема
В обзоре admin-панели показывается счётчик «Уволенных: 2», но нельзя посмотреть кто именно.

### Что сделать

**Фронтенд** `frontend/src/app/admin/page.tsx`:

Карточка «УВОЛЕННЫХ» в обзоре — сделать кликабельной:
- Клик → открывает модальное окно или раскрывающийся блок под карточкой
- Список уволенных: ФИО / Должность / Подразделение / Дата увольнения
- Стиль: `.cyber-card`, строки таблицы в тёмном стиле
- Бейдж `badge-danger` «Уволен» у каждой строки
- Кнопка закрытия модального окна

**Бэкенд** `backend/app/api/admin.py`:

Добавить эндпоинт:
```
GET /api/admin/employees/dismissed
```

Возвращает:
```json
[
  {
    "redmine_id": 123,
    "name": "Иванов Иван",
    "position_id": "15",
    "role_id": "ПРА_КОН_015",
    "role_name": "Консультант",
    "unit": "Правовое управление",
    "dismissed_at": "2026-04-20"
  }
]
```

Источник: `employees` где `status = 'dismissed'`
