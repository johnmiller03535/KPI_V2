## КОНТЕКСТ

Таб «Подчинённость» в `/admin` после редизайна имеет 4 проблемы:
1. Сохранение руководителя падает с `[Errno 30] Read-only file system`
2. `has_kpi_card` возвращает `false` для всех 91 должности (джойн сломан)
3. Дерево — просто сгруппированный список, нужна левая панель с подразделениями
4. Кнопка «Обновить» бесполезна, бейджи в списке и дереве разные

---

## ЗАДАЧА 1 — Сохранять подчинённость в PostgreSQL (не в json-файл)

### Проблема
При нажатии «Изменить» → сохранить руководителя backend пытается записать в
`/app/reference/subordination.json`, который смонтирован **read-only**.

Ошибка: `[Errno 30] Read-only file system: '/app/reference/subordination.json'`

### Решение
Таблица `subordination` в PostgreSQL уже существует (создавалась ранее).
Нужно переключить все операции записи с файла на таблицу.

**Backend — `backend/app/api/admin.py`:**

Эндпоинт `PATCH /api/admin/subordination/{position_id}` должен:
```python
# БЫЛО: открывать и перезаписывать subordination.json
# СТАЛО: UPDATE в таблицу subordination
await db.execute(
    update(Subordination)
    .where(Subordination.position_id == position_id)
    .values(evaluator_id=new_evaluator_id)
)
await db.commit()
```

**Чтение подчинённости** (`_get_effective_subordinate_ids` в `review.py`):
- Сначала читать из таблицы `subordination`
- Если таблица пуста — fallback на `subordination.json` (для обратной совместимости)

**Импорт из People** (`POST /api/admin/subordination/import`):
- Данные импортировать в таблицу `subordination`, не в файл
- Файл `subordination.json` оставить как readonly-справочник, не трогать

### Миграция
Проверить что таблица `subordination` существует и имеет нужные колонки:
```sql
position_id   VARCHAR  -- код должности (РУК_ЗАМД_002)
evaluator_id  VARCHAR  -- код руководителя (null = директор)
```
Если таблица пуста — при первом обращении автоматически заполнить из `subordination.json`.

## ЗАДАЧА 2 — Исправить `has_kpi_card` (всегда возвращает false)

### Проблема
В табе подчинённости все 91 должность показывает `⚠️ нет карточки`, хотя
в `kpi_role_cards` импортированы 91 карточка.

Причина: джойн использует неправильный ключ. Скорее всего сравнивается
`subordination.position_id` (например `"РУК_ЗАМД_002"`) с `kpi_role_cards.position_id`
(например `"4"` числовой pos_id) — они не совпадают.

### Решение
Сначала проверить через SQL что реально хранится в таблицах:
```sql
SELECT position_id, position_name FROM kpi_role_cards LIMIT 5;
SELECT position_id FROM subordination LIMIT 5;
SELECT position_code, pos_id FROM users LIMIT 5;
```

Скорее всего нужно джойнить через таблицу `users`:
```
subordination.position_id = "РУК_ЗАМД_002"  (строковый код)
users.position_code       = "РУК_ЗАМД_002"  (тот же код)
users.pos_id              = 4               (числовой id)
kpi_role_cards.position_id = "4"            (строка с числом)
```

Исправить джойн в `GET /api/admin/subordination` — вычисление поля `has_kpi_card`.

## ЗАДАЧА 3 — Дерево: переделать на левую панель + правая часть

### Нужно
Двухколоночный layout аналогично табу «KPI-показатели»:

```
[Левая панель]              [Правая часть]
────────────────            ───────────────────────────────────────
Все должности  91           Первый зам. директора
─────────────────           РУК_ПЕРЗ_001  → директор
Руководство     4           ⚠️ нет карточки          [Изменить]
Уп. орг. обесп 12
ЕАС            13           Заместитель директора
КЗА             8           РУК_ЗАМД_002  → директор
...                         ⚠️ нет карточки          [Изменить]
─────────────────
⚠️ Без карточки 91
```

**Поведение:**
- По умолчанию выбрано «Все должности» — показывает всех 91
- Клик на подразделение → фильтрует правую часть
- Клик «⚠️ Без карточки» → только должности без карточки
- Бейдж `⚠️ нет карточки` кликабельный → переход в таб «KPI-карточки»

**Режим «Список»** — старая таблица, без изменений кроме унификации бейджей (задача 4).

## ЗАДАЧА 4 — Убрать «Обновить», унифицировать бейджи

### Убрать кнопку «Обновить»
Бесполезна для пользователя. Убрать полностью. Остаётся только `[Импорт из People]`.

### Единый стандарт бейджей везде (список + дерево)
- Карточка есть: `✅ карточка` — зелёный (`--accent3 #00ff9d`)
- Карточки нет: `⚠️ нет карточки` — жёлтый (`--warn #ffb800`), кликабельный → KPI-карточки
