## ЧАСТЬ 1 — ИСПРАВИТЬ ИМПОРТ (критично)

### Проблема
В БД только 1 общий показатель вместо 3:
```sql
SELECT name, is_common FROM kpi_indicators WHERE is_common=true;
-- Возвращает 1 строку вместо 3
```

Три общих показателя в xlsx:
1. «Соблюдение исполнительской дисциплины (МСЭД)» — 30%, binary_manual
2. «Соблюдение Правил трудового распорядка, Кодекса этики» — 10%, binary_manual  
3. «Соблюдение правил техники безопасности, охраны труда» — 10%, binary_manual

В xlsx все три имеют `indicator = "Общие показатели эффективности и результативности деятельности"` но разные `criterion`. Импорт слил их в один объект.

### Исправление в kpi_import_service.py

Уникальность показателя определять по комбинации `(indicator + criterion)`, а не только по `indicator`:

```python
# Было: ключ = indicator_name
indicator_key = row['indicator']

# Стало: ключ = indicator_name + criterion
indicator_key = f"{row['indicator']}||{row['criterion']}"
```

После исправления:
1. Очистить таблицы: `TRUNCATE kpi_indicators, kpi_criteria, kpi_role_cards, kpi_role_card_indicators CASCADE;`
2. Повторить импорт: `POST /api/kpi/import/xlsx`
3. Проверить: `SELECT name, is_common FROM kpi_indicators WHERE is_common=true;` — должно быть 3 строки

### Редактор показателя

При клике «Редактировать» — модальное окно или страница с формой.
Форма зависит от `formula_type`.

#### Общие поля (все типы)
```
Название:         [textarea]
Тип:              [dropdown - менять нельзя после создания]
Общий показатель: [checkbox - только hr/admin]
Группа:           [dropdown из INDICATOR_GROUPS]
```

#### binary_auto / binary_manual
```
Критерий оценки: [textarea]
Порог выполнено:     [текст] → [оценка %]  (например "Выполнено → 100")
Порог не выполнено:  [текст] → [оценка %]  (например "Не выполнено → 0")
Для is_common:
  Текст при выполнении:     [textarea]
  Текст при невыполнении:   [textarea]
```

#### threshold
```
Критерий оценки:  [textarea]
Числитель:        [input] — что вводит сотрудник
Знаменатель:      [input] — что вводит сотрудник
Нарастающим итогом: [checkbox]
Целевое значение: [input] — текст (например ">=67%")
Пороги:           [динамический список]
  + Добавить порог
  Условие: [input] (например ">=67")  Оценка: [input %]
  Условие: [input] (например ">=50")  Оценка: [input %]
  [🗑] для каждого порога
```

#### multi_threshold
```
Критерий оценки: [textarea]
Подпоказатели:   [динамический список]
  + Добавить подпоказатель
  ┌────────────────────────────────────────────┐
  │ Название: [input]                          │
  │ Числитель: [input]                         │
  │ Знаменатель: [input]                       │
  │ Порог: [условие] → [оценка]                │
  │ [🗑 Удалить подпоказатель]                 │
  └────────────────────────────────────────────┘
Логика: невыполнение одного = невыполнение всего (автоматически)
```

#### quarterly_threshold
```
Критерий оценки: [textarea]
Числитель:       [input]
Знаменатель:     [input]
Пороги по кварталам:
  Q1: Условие [input]  Оценка [input %]
  Q2: Условие [input]  Оценка [input %]
  Q3: Условие [input]  Оценка [input %]
  Q4: Условие [input]  Оценка [input %]
Нарастающим итогом: [checkbox]
```

### Кнопки в редакторе
- «Сохранить» → PUT /api/kpi/indicators/{id}, статус сразу active
- «Отмена» → закрыть без сохранения
- «Создать новый» → POST /api/kpi/indicators, пустая форма

### Отображение карточки

Таблица показателей с весами:
```
№ | Название показателя | Тип | Группа | Вес | Общий | Действия
1 | Обеспечение проектной деятельности | Авто | Проектная | 20% | - | [↑][↓][✏️][🗑]
2 | Соблюдение дисциплины (МСЭД) | Ручной | Общий | 30% | ✅ | [↑][↓][-][-]
```

Внизу: **Сумма весов: 100%** (зелёный) / **Сумма весов: 85%** (красный)

### Редактирование карточки (только admin)

**Добавить показатель:**
- Кнопка «+ Добавить показатель»
- Модалка с поиском по библиотеке показателей
- Ввод веса для добавляемого показателя
- Валидация: сумма весов не превышает 100%

**Изменить вес:**
- Клик на вес → inline редактирование
- Сохранение по blur или Enter

**Удалить показатель:**
- Кнопка 🗑 (только для не-общих показателей)
- Подтверждение: «Удалить показатель из карточки?»

**Изменить порядок:**
- Кнопки ↑↓ для перемещения строк

**Сохранить карточку:**
- Кнопка «Сохранить изменения» — PUT /api/kpi/cards/{id}/indicators
- Валидация суммы весов = 100% перед сохранением

## API ИЗМЕНЕНИЯ

### Добавить поле `unit` в GET /api/kpi/cards
```python
# В ответе каждой карточки добавить:
"unit": role_card.unit  # подразделение из KPI_Mapping
```

### Добавить поле `group` в kpi_indicators
```sql
ALTER TABLE kpi_indicators ADD COLUMN indicator_group VARCHAR;
```

При импорте — автоматически классифицировать по INDICATOR_GROUPS.
В редакторе — можно изменить вручную.

### PUT /api/kpi/indicators/{id} — полное обновление
Принимать все поля включая criteria (thresholds, sub_indicators, etc.)
Сохранять сразу как active.
