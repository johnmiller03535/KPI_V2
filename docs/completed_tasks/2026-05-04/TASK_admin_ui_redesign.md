## КОНТЕКСТ

В admin-панели (`/admin`) сейчас 6 табов. Три из них требуют доработки:

- **«Подчинённость»** — плоский список 91 строки, неудобно
- **«KPI-карточки»** — одинокий дропдаун без обзора
- **«KPI-показатели»** — нет кнопки «+ Добавить показатель»

Главный принцип: **Redmine — источник истины** для структуры организации.  
Портал только _обогащает_ структуру (KPI-карточки, подчинённость для review).

## ЗАДАЧА 1 — Таб «Подчинённость»: добавить вид «Дерево»

### Что сделать

Добавить переключатель **[Список] / [Дерево]** в правом верхнем углу таба (рядом с кнопками «Обновить» и «Импорт из People»).

**Вид «Список»** — оставить как есть (текущая таблица).

**Вид «Дерево»** — иерархическое дерево, сгруппированное по подразделениям:

```
📁 Руководство
   ── 👤 Директор  [вне системы]
      ├── 👤 Первый зам. директора  РУК_ПЕРЗ_001  [карточка ✅]  ✏️ Изменить
      ├── 👤 Заместитель директора  РУК_ЗАМД_002  [карточка ✅]  ✏️ Изменить
      ├── 👤 Заместитель директора  РУК_ЗАМД_003  [карточка ✅]  ✏️ Изменить
      └── 👤 Заместитель директора  РУК_ЗАМД_004  [карточка ✅]  ✏️ Изменить

📁 Управление организационного обеспечения...
   ── 👤 Начальник управления  ОРГ_НАЧ_005
      ├── 👤 Заместитель начальника управления  ОРГ_ЗАМ_006  [карточка ✅]  ✏️
      └── 👤 Зам. нач. упр. — нач. отдела  ОРГ_ЗАМ_НАЧ_ОТД_007  [карточка ✅]  ✏️
```

### Статус карточки рядом с каждой должностью

- `[карточка ✅]` — зелёный бейдж — карточка есть
- `[⚠️ нет карточки]` — жёлтый бейдж — карточки нет, кликабельный → переход в таб KPI-карточки с этой должностью предвыбранной

### API для дерева

Использовать существующий `GET /api/admin/subordination` — данные уже есть.  
Для статуса карточек добавить поле `has_kpi_card: bool` в ответ эндпоинта
`GET /api/admin/subordination` (джойн с таблицей `kpi_role_cards` по `position_id`).

**Backend:** `backend/app/api/admin.py` — в роутере subordination добавить джойн.

## ЗАДАЧА 2 — Таб «KPI-карточки»: сайдбар по подразделениям

### Что сделать

Полностью переделать таб «KPI-карточки» — убрать дропдаун, сделать двухколоночный layout как в табе «KPI-показатели».

**Левый сайдбар** — подразделения с количеством должностей:

```
Все карточки              91
─────────────────────────────
Руководство                4  ← название подразделения
Уп. орг. обеспечения...   12  ← счётчик должностей в нём
Уп. ЕАС                   13
КЗА                        8
...
⚠️ Без карточки            2  ← должности из Redmine без карточки
```

**Правая часть** — при выборе подразделения показывает список должностей:

```
[+ Создать карточку]                          ← кнопка вверху

┌─────────────────────────────────────────────────────┐
│ Заместитель директора  pos_id=4 · v1 · active       │
│ Руководство                              [Открыть →] │
└─────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────┐
│ ⚠️ Новая должность (без карточки)                   │
│ Руководство               [Создать карточку →]       │
└─────────────────────────────────────────────────────┘
```

При клике «Открыть →» / «Создать карточку →» — разворачивается та же форма, что сейчас работает (список показателей + редактирование весов).

### Секция «⚠️ Без карточки»

Должности, которые есть в таблице `users` (из Redmine), но у которых нет записи в `kpi_role_cards`. Это сигнал admin'у что нужно создать карточку.

**API:** добавить `GET /api/admin/kpi-cards/positions-without-cards` → список должностей из `users` не имеющих карточки.

## ЗАДАЧА 3 — Wizard «Создать карточку»

### Что сделать

При нажатии «+ Создать карточку» → открывается модальное окно (2 шага):

**Шаг 1 — Основные данные:**
```
Название должности:  [_________________________]
Подразделение:       [выпадающий список из существующих групп ▼]
Код должности:       [авто: ПРЕ_ДОЛ_092 ▼ или ввод вручную]
```

**Шаг 2 — Источник показателей:**
```
○ Создать пустую карточку (добавлю показатели сам)
○ Скопировать с существующей должности:
     [выпадающий список карточек ▼]
     
                    [Отмена]  [Создать карточку →]
```

После создания — карточка сразу открывается в режиме редактирования.

### Backend

`POST /api/admin/kpi-cards/` — уже должен существовать, проверить и дополнить если нужно:
```json
{
  "position_name": "Специалист",
  "department": "Руководство",
  "position_id": "РУК_СПЦ_092",
  "copy_from_card_id": 4  // null если пустая
}
```

## ЗАДАЧА 4 — Таб «KPI-показатели»: кнопка «+ Добавить показатель»

### Что сделать

Добавить кнопку `[+ Добавить показатель]` в правом верхнем углу (рядом со строкой поиска).

При клике — та же модальная форма что «Редактировать», но:
- поля пустые
- добавлен `<select>` для выбора типа в начале формы: `binary_auto / binary_manual / threshold / multi_threshold / quarterly_threshold`
- при смене типа — форма адаптируется (threshold показывает числитель/знаменатель/пороги, binary — только критерий)

**API:** `POST /api/admin/indicators/` — проверить что существует.

### Также исправить счётчик в сайдбаре

Сейчас показывает **124** вместо **126**. Проверить запрос — возможно фильтруются `is_active=false` записи. Показывать все, включая неактивные (с визуальным отличием).

## ВАЖНЫЕ ОГРАНИЧЕНИЯ

1. **Не трогать** логику `kpi_role_card_indicators` — только UI поверх существующих данных
2. **Не удалять** дропдаун полностью — можно переиспользовать внутри wizard

4. Создание новых должностей — **только через Redmine + Импорт**, не через портал напрямую
