## УЛУЧШЕНИЕ 1 — Выбор должности из структуры (вместо ручного ввода pos_id)

### Проблема
Сейчас wizard Шаг 1 просит ввести `pos_id` вручную — пользователь не знает id,
вводит произвольное число и получает чужие данные.

### Решение — иерархический выбор должности

Заменить поля «Название должности» + «POS_ID» + «Подразделение» на
**трёхуровневый селектор из таблицы subordination**:

```
ШАГ 1 из 2 — Выбор должности

УПРАВЛЕНИЕ *
[Выберите управление ▼]
  Руководство
  Правовое управление
  Управление анализа и автоматизации данных
  ...

ПОДРАЗДЕЛЕНИЕ (если есть отделы внутри)
[Выберите подразделение ▼]  ← появляется после выбора управления
  Отдел судебной практики
  Отдел нормативного сопровождения
  Показать всё управление (без отдела)

ДОЛЖНОСТЬ *
[Выберите должность ▼]  ← появляется после выбора подразделения
  Начальник управления
  Заместитель начальника управления
  Консультант
  ...

После выбора должности автоматически заполняются:
  Название: [Консультант]                    ← из subordination.role
  POS_ID:   [ПРА_КОН_036]                   ← из subordination.role_id
  Подразделение: [Правовое управление]       ← из subordination.unit

  ✓ Карточка ещё не создана для этой должности  ← зелёный
  ⚠️ Карточка уже существует для pos_id=36    ← жёлтый с предупреждением
```

### Логика группировки для селектора управлений
Использовать `normalizeUnit()` из `frontend/src/utils/admin.ts` —
та же что в сайдбаре KPI-карточек и Подчинённости.

### API
`GET /api/admin/subordination` уже возвращает все данные включая role_id, unit, role.
Дополнительных запросов не нужно — использовать уже загруженные данные.

### Если карточка уже существует
Показать предупреждение: «Карточка для этой должности уже существует.
Хотите открыть существующую карточку?» с кнопкой «Открыть».

## УЛУЧШЕНИЕ 2 — Автоматически добавлять общие показатели в новую карточку

### Проблема
Три общих показателя (`is_common=true`) должны быть в карточке каждого сотрудника.
Сейчас их нужно добавлять вручную.

### Решение
При создании новой карточки (в любом варианте — пустая или копия):
1. Автоматически добавить все `is_common=true` показатели
2. С их стандартными весами: исполнительская дисциплина=30, трудовой распорядок=10, охрана труда=10
3. Итого автоматически 50% — пользователь добавляет специфические показатели на оставшиеся 50%

**Backend** — в эндпоинте `POST /api/admin/kpi-cards/`:
```python
# После создания карточки — добавить is_common показатели
common_indicators = await db.execute(
    select(KpiIndicator).where(KpiIndicator.is_common == True, KpiIndicator.status == 'active')
)
for ind in common_indicators.scalars():
    # Стандартный вес из kpi_criteria или захардкодить 30/10/10
    weight = DEFAULT_COMMON_WEIGHTS.get(ind.id, 10)
    db.add(KpiRoleCardIndicator(card_id=new_card.id, indicator_id=ind.id, weight=weight))
```

Стандартные веса общих показателей:
```python
# Определить по названию или хранить в kpi_indicators.default_weight
# Исполнительская дисциплина → 30
# Трудовой распорядок → 10
# Охрана труда → 10
```

Добавить поле `default_weight INTEGER DEFAULT NULL` в `kpi_indicators` —
хранить дефолтный вес для `is_common` показателей.

**Frontend** — после создания карточки показать сообщение:
«Карточка создана. Автоматически добавлены 3 общих показателя (50%). Добавьте специфические показатели.»

При копировании карточки — is_common показатели берутся из источника (они там уже есть).
