## БАГ 1 — Новые показатели создаются со статусом `draft` вместо `active`

### Проблема
Все созданные вручную показатели имеют статус `draft` — они не видны в диалоге
добавления в карточку и не добавляются автоматически как is_common.

### Причина
В API `POST /api/admin/indicators/` статус по умолчанию = `draft`.

### Решение
Изменить дефолтный статус на `active` при создании через admin:

```python
# backend/app/api/admin.py или kpi_constructor.py
# При создании показателя:
ind = KpiIndicator(
    ...
    status='active',  # было 'draft', стало 'active'
)
```

**Обоснование:** показатели создаёт начальник управления или admin.
Workflow draft→review→active будет реализован позже.
Сейчас все новые показатели должны сразу быть активными.

## БАГ 2 — `unit_name` не сохраняется при создании/редактировании

### Проблема
Поле «УПРАВЛЕНИЕ» в форме показателя не сохраняется в БД.

### Диагностика
Проверить в `backend/app/api/admin.py` эндпоинт `POST /indicators/`
и `PATCH /indicators/{id}` — передаётся ли `unit_name` в модель:

```python
# Должно быть:
ind = KpiIndicator(
    ...
    unit_name=body.unit_name,  # ← это поле передаётся?
)
```

Также проверить что поле `unit_name` есть в схеме `IndicatorCreate` и `IndicatorUpdate`.

## БАГ 3 — Диалог «Добавить показатель» в карточке

### Проблемы
1. Показывает только активные (`status='active'`) — это правильно, но нужно проверить
2. Фильтр только по группам — нужно добавить фильтр по управлению (`unit_name`)
3. Не загружает показатели (возможно API фильтрует по статусу и draft не показывает)

### Решение

**3.1 Фильтр по управлению**

Добавить второй select рядом с «Все группы»:

```tsx
<div style={{ display: 'flex', gap: 8, marginBottom: 12 }}>
  <input placeholder="Поиск по названию..." />
  <select> {/* Все управления */}
    <option value="">Все управления</option>
    {uniqueUnits.map(u => <option key={u} value={u}>{u}</option>)}
  </select>
  <select> {/* Все группы */}
    <option value="">Все группы</option>
    {groups.map(g => <option key={g} value={g}>{g}</option>)}
  </select>
</div>
```

Фильтрация: сначала по управлению (`unit_name`), потом по группе, потом по поиску.

**3.2 Показывать только `active` показатели**

API для диалога должен возвращать только `status='active'`.
Убедиться что запрос не включает `draft`.

**3.3 Автодобавление is_common при создании карточки**

Если карточка создана до того как появились is_common показатели —
при открытии карточки в режиме редактирования автоматически добавить
отсутствующие is_common показатели.

Или добавить кнопку «Синхронизировать общие показатели» в шапке карточки.

## БАГ 4 — Общие показатели не добавляются автоматически если карточка создана раньше

### Решение
В эндпоинте `GET /api/admin/kpi-cards/{id}` или при открытии карточки:
проверять наличие всех is_common показателей и добавлять отсутствующие.

Или проще — добавить API эндпоинт:
```
POST /api/admin/kpi-cards/{id}/sync-common
```
Добавляет все is_common active показатели в карточку если их ещё нет.

И кнопку в UI карточки: «+ Добавить общие показатели» (показывать только если
хотя бы один is_common отсутствует).
