## БАГ 1 — Кнопки «Отмена / Создать показатель» перекрывают поля формы

### Проблема
Кнопки в футере модалки накладываются поверх полей формы (особенно при скролле
внутри модалки). Пользователь не может заполнить нижние поля.

### Решение
Футер с кнопками должен быть **зафиксирован внизу модалки** — sticky внутри контейнера,
не скроллится вместе с содержимым:

```tsx
// Структура модалки:
<div className="modal-content" style={{
  display: 'flex',
  flexDirection: 'column',
  maxHeight: '88vh',
}}>
  {/* Скроллящаяся часть */}
  <div style={{ flex: 1, overflowY: 'auto', padding: '24px 24px 0' }}>
    {/* Все поля формы */}
  </div>

  {/* Зафиксированный футер */}
  <div style={{
    padding: '16px 24px',
    borderTop: '1px solid rgba(0,229,255,0.2)',
    display: 'flex',
    justifyContent: 'flex-end',
    gap: '12px',
    background: 'var(--card)',
    flexShrink: 0,
  }}>
    <button onClick={onClose}>Отмена</button>
    <button onClick={handleSubmit}>+ Создать показатель</button>
  </div>
</div>
```

## БАГ 2 — Группа не сохраняется при создании (сбрасывается в «Прочие»)

### Проблема
Пользователь выбирает группу в дропдауне при создании нового показателя,
но после сохранения показатель оказывается в «Прочие показатели».
При редактировании и повторном сохранении — группа сохраняется корректно.

### Причина
Скорее всего начальное значение `group` в state формы инициализируется как
`'Прочие показатели'` и не обновляется при выборе из дропдауна —
либо в `handleSubmit` отправляется дефолтное значение вместо выбранного.

### Диагностика
```typescript
// Проверить: при onChange дропдауна группы
// state.group должен обновляться
console.log('Selected group:', selectedGroup) // что отправляется в API?

// Проверить body POST-запроса в Network tab браузера
// поле group_name должно совпадать с выбранным
```

### Исправление
Убедиться что:
1. `onChange` дропдауна обновляет правильный ключ в state
2. `handleSubmit` читает актуальное значение из state, не дефолт
3. Начальное значение `group` = `''` или `null`, а не `'Прочие показатели'`

## БАГ 3 — Модалка перекрывается навбаром сверху

### Проблема
Верхняя часть модалки уходит под навбар (top: 64px занят).

### Исправление
Overlay должен быть `position: fixed; inset: 0` и центрировать модалку
с учётом высоты навбара:

```css
.modal-overlay {
  position: fixed;
  inset: 0;
  display: flex;
  align-items: center;    /* центр по вертикали */
  justify-content: center;
  z-index: 100;           /* выше навбара (z-index навбара ~30) */
  background: rgba(0,0,0,0.75);
  padding-top: 0;         /* НЕ добавлять отступ сверху */
}
```

Если навбар имеет `position: sticky; z-index: 30` — overlay с `z-index: 100`
будет поверх него автоматически.
