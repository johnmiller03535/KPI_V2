# Задача: Улучшить промпт генерации саммари
> Файл: backend/app/services/kpi_engine_service.py

## Новый промпт для generate_summary_from_tasks

```python
# Агрегировать задачи по названию
from collections import Counter
task_counts = Counter(e['issue']['subject'] for e in work_entries)

# Формировать список с количеством
tasks_text = "\n".join(
    f"- {task} (×{count})" if count > 1 else f"- {task}"
    for task, count in task_counts.most_common()
)

prompt = f"""Ты — система подготовки отчётов KPI государственного учреждения.
Составь формальное деловое саммари выполненных работ сотрудника.

Должность: {role_name}
Показатель KPI: {indicator.indicator}
Критерий: {indicator.criterion}

Задачи из Redmine (×N = количество раз за период):
{tasks_text}

{absence_note}

Правила составления саммари:
1. Сгруппируй задачи по смыслу: разработка/автоматизация, аналитика/отчётность, совещания/координация, защиты/презентации
2. Для повторяющихся задач указывай количество: "5 раз", "3 итерации", "4 совещания"
3. Деловой стиль — глаголы совершенного вида: "Выполнена", "Проведена", "Подготовлены", "Обеспечено участие"
4. Период НЕ упоминать
5. Пиши столько предложений сколько нужно чтобы полно отразить все виды работ — не сокращай
6. Только факты из списка — не придумывай
7. Если список пуст — "Рабочие задачи за период не зафиксированы"

Пример хорошего саммари:
"Выполнена разработка и доработка модулей формирования отчётов (6 итераций), 
настройка системы Redmine (3 раза). Проведена подготовка аналитических данных (5 раз), 
обновление показателей дашборда (4 раза). Обеспечено участие в совещаниях 
по проектам трансформации (5), оперативных совещаниях ЗПМО (4)."
"""
```

## Чеклист
- [ ] Задачи агрегированы с подсчётом (Counter)
- [ ] Период не упоминается в тексте саммари
- [ ] Повторяющиеся задачи указаны с количеством
- [ ] Нет ограничения на количество предложений
- [ ] Пересобрать бэкенд и проверить
