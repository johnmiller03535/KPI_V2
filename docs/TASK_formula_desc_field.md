# ЗАДАЧА: Добавить поле «Методика расчёта» в конструктор показателей

> Файл для Claude Code | Дата: 2026-05-04 | Приоритет: 🟡

---

## Проблема

В форме редактирования показателей не отображается поле `formula_desc` из таблицы `kpi_criteria`.
Это поле содержит описание методики в человекочитаемом виде.

Пример:
- Критерий оценки: «Обеспечение качественного согласования проектов ЛНА и (или) НПА»
- **Методика расчёта:** «Отсутствие более 2-х повторных согласований проектов ЛНА и (или) НПА» ← не отображается
- Пороги: `<=2 → 100 | >2 → 0`

---

## Решение

### 1. Проверить наличие поля в БД
```sql
SELECT column_name FROM information_schema.columns
WHERE table_name = 'kpi_criteria' AND column_name = 'formula_desc';
```

Если поля нет — добавить через миграцию:
```sql
ALTER TABLE kpi_criteria ADD COLUMN IF NOT EXISTS formula_desc TEXT DEFAULT NULL;
```

### 2. Добавить поле в API схемы
В `backend/app/schemas/admin.py` добавить `formula_desc: str | None = None`
в схемы `CriterionCreate`, `CriterionUpdate`, `CriterionResponse`.

### 3. Добавить поле в форму конструктора

Показывать для типов: `threshold`, `multi_threshold`, `quarterly_threshold`, `absolute_threshold`.
Для `binary_*` — не показывать (там нет числовой методики).

Расположение — между «КРИТЕРИЙ ОЦЕНКИ» и блоком «ЧИСЛОВОЙ ПОКАЗАТЕЛЬ»:

```
КРИТЕРИЙ ОЦЕНКИ *
[textarea — общее описание]

МЕТОДИКА РАСЧЁТА
[textarea — как именно считается, что считается выполнением]
Подсказка: Опишите условие выполнения показателя в понятном виде.
           Пример: «Отсутствие более 2-х повторных согласований ЛНА»,
                   «Доля конкурентных закупок не менее 67% от общего объёма»

ЧИСЛОВОЙ ПОКАЗАТЕЛЬ
[числитель / знаменатель / пороги...]
```

### 4. Отображать в модалке «Просмотр»

Если `formula_desc` заполнен — показывать между критерием и порогами:
```
КРИТЕРИЙ ОЦЕНКИ
[текст]

МЕТОДИКА РАСЧЁТА
[текст — выделить курсивом или приглушённым цветом]

ПОРОГИ
[список]
```

### 5. Отображать в форме сотрудника (шаг K)
При заполнении KPI-формы сотрудник должен видеть `formula_desc` рядом с полем ввода,
чтобы понимать что именно вводить.
