## 🔵 ТАБ «ПОДЧИНЁННОСТЬ»

### Переключатель Список / Дерево
- [ ] Переключатель отображается рядом с кнопками «Обновить» и «Импорт из People»
- [ ] По умолчанию активен «Список» (старое поведение не сломано)
- [ ] Переключение «Дерево» работает без перезагрузки страницы
- [ ] Состояние переключателя сохраняется при смене таба и возврате

### Вид «Список» (не должен измениться)
- [ ] 91 строка отображается как прежде
- [ ] Кнопка «Изменить» работает, сохранение работает
- [ ] «Обновить» и «Импорт из People» работают

### Вид «Дерево»
- [ ] Должности сгруппированы по подразделениям
- [ ] Иерархия корректна (начальники → заместители → специалисты)
- [ ] Узлы раскрываются/сворачиваются кликом на подразделение
- [ ] Отступы визуально показывают уровень иерархии
- [ ] У каждой должности есть бейдж статуса карточки

### Бейджи статуса карточки в дереве
- [ ] `✅ карточка` — зелёный — отображается у должностей с карточкой
- [ ] `⚠️ нет карточки` — жёлтый — отображается у должностей без карточки
- [ ] Клик на `⚠️ нет карточки` → переход в таб KPI-карточки с предвыбором этой должности
- [ ] Кнопка «Изменить» в дереве работает (открывает модалку смены руководителя)

### Баннер после импорта
- [ ] После «Импорт из People» — если есть должности без карточек — показывается баннер
- [ ] Баннер содержит список конкретных должностей (не просто счётчик)
- [ ] Кнопка «Перейти в KPI-карточки →» работает
- [ ] Если все должности с карточками — баннер не показывается

## 🔵 ТАБЛИЦЫ KPI-КАРТОЧКИ

### Сайдбар по подразделениям
- [ ] Вместо дропдауна — левый сайдбар с подразделениями
- [ ] Счётчик должностей рядом с каждым подразделением корректен
- [ ] Пункт «Все карточки» показывает все 91
- [ ] Пункт «⚠️ Без карточки» присутствует и показывает нужные должности
- [ ] Активное подразделение выделено акцентным цветом (Dark Cyber стиль)

### Список должностей в правой части
- [ ] При выборе подразделения — список его должностей отображается
- [ ] У каждой карточки видны: название, pos_id, версия, статус, подразделение
- [ ] Должности без карточки показываются с пометкой `⚠️ новая`
- [ ] Кнопка «Открыть →» работает — разворачивает карточку для просмотра/редактирования
- [ ] Кнопка «Создать карточку →» видна только у должностей без карточки

### Редактирование карточки (существующий функционал — не должен сломаться)
- [ ] Список показателей отображается корректно
- [ ] Веса редактируются
- [ ] Сумма весов считается (100% = «✓ Верно»)
- [ ] «+ Добавить показатель» в карточке работает
- [ ] Кнопки «Сохранить» / «Отмена» работают
- [ ] Стрелки сортировки (▲▼) работают
- [ ] Корзина (удалить показатель из карточки) работает

## 🔵 WIZARD «СОЗДАТЬ КАРТОЧКУ»

### Открытие и навигация
- [ ] Кнопка `[+ Создать карточку]` видна в правом верхнем углу таба
- [ ] Кнопка открывает модальное окно (не новую страницу)
- [ ] Шаг 1 отображается первым
- [ ] Кнопка «Отмена» закрывает без сохранения
- [ ] ESC закрывает модалку

### Шаг 1 — Основные данные
- [ ] Поле «Название должности» — обязательное, валидация при пустом
- [ ] Выпадающий список подразделений заполнен (берёт из существующих групп)
- [ ] Поле «Код должности» — автогенерация или ввод вручную
- [ ] «Далее →» не активна пока не заполнены обязательные поля

### Шаг 2 — Источник показателей
- [ ] Радиокнопки: «Пустая карточка» / «Скопировать с...»
- [ ] При выборе «Скопировать» — появляется дропдаун со списком карточек
- [ ] Дропдаун карточек заполнен всеми 91 карточкой
- [ ] Кнопка «Создать карточку →» активна
- [ ] После создания — карточка сразу открывается в режиме редактирования
- [ ] При копировании — показатели скопированы с источника (с теми же весами)
- [ ] При создании пустой — карточка пуста, можно сразу добавлять показатели

### Edge cases wizard
- [ ] Нельзя создать карточку с дублирующим pos_id → ошибка
- [ ] Нельзя создать карточку с пустым именем → ошибка на фронте
- [ ] Если API вернул ошибку — показывается понятное сообщение

## 🔵 ТАБЛИЦА KPI-ПОКАЗАТЕЛИ

### Кнопка «+ Добавить показатель»
- [ ] Кнопка присутствует рядом со строкой поиска
- [ ] Открывает модальное окно создания
- [ ] В начале формы — `<select>` выбора типа
- [ ] При выборе `binary_auto` / `binary_manual` — показывается только поле «Критерий оценки» и «Группа»
- [ ] При выборе `threshold` — дополнительно показываются «Числитель», «Знаменатель», «Пороги (JSON)», чекбокс «Нарастающим итогом»
- [ ] При выборе `multi_threshold` — то же что threshold
- [ ] При выборе `quarterly_threshold` — поле пороги с подсказкой о квартальной структуре
- [ ] После сохранения — новый показатель появляется в списке
- [ ] После сохранения — счётчик в сайдбаре увеличивается

### Счётчик показателей
- [ ] «Все» показывает **126** (не 124)
- [ ] Неактивные показатели (`is_active=false`) отображаются с визуальным отличием (например, приглушённый цвет)
- [ ] Фильтр по группе работает корректно после исправления счётчика
