# ЗАДАЧА: Разобраться почему счётчик показателей 124 вместо 126

> Файл для Claude Code | Дата: 2026-05-02 | Приоритет: 🟡

---

## Проблема

API `/api/admin/indicators/?status=all` возвращает 124 записи вместо 126.
В БД после импорта должно быть 126. Фронтенд считает правильно — проблема в backend.

## Диагностика

Выполнить в контейнере backend или через Swagger:

```sql
-- Сколько всего записей в таблице?
SELECT COUNT(*) FROM kpi_indicators;

-- Есть ли записи с нестандартным статусом?
SELECT status, COUNT(*) FROM kpi_indicators GROUP BY status;

-- Есть ли deleted/archived записи?
SELECT COUNT(*) FROM kpi_indicators WHERE is_active = false;
SELECT COUNT(*) FROM kpi_indicators WHERE status NOT IN ('active', 'inactive');
```

## Вероятные причины

**Вариант А** — в таблице реально 124 записи (2 не импортировались)
→ Проверить логи импорта, найти какие 2 показателя пропущены

**Вариант Б** — в таблице 126, но API фильтрует по какому-то полю
→ Проверить роутер `GET /api/admin/indicators/` на скрытые фильтры
→ Убедиться что `?status=all` реально отключает все фильтры

**Вариант В** — пагинация обрезает результат
→ Проверить есть ли `limit` по умолчанию в эндпоинте (например limit=100)
→ Если есть — увеличить default limit или добавить `?limit=200`

## Исправление

После нахождения причины — починить так чтобы все 126 возвращались.
Если проблема в импорте — перезапустить импорт и проверить логи.
