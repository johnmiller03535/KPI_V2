## ЗАДАЧА 1 — Левая панель: дедупликация и правильный порядок

### Проблема
«Управление организационного обеспечения, бюджетного учета и финансовой отчетности»
появляется в панели 5+ раз с разными счётчиками (5, 1, 1, 1, 3, 2, 1...).

Причина: строки после `split('.')[0].trim()` технически разные из-за
невидимых символов, двойных пробелов или разных кодировок пробела.

### Решение

**Шаг 1 — Жёсткая нормализация строк перед группировкой:**
```typescript
function normalizeUnit(dept: string): string {
  return dept
    .split('.')[0]          // берём до первой точки
    .replace(/\s+/g, ' ')   // все множественные пробелы → один
    .trim()                 // обрезаем края
}
```

**Шаг 2 — Группировка через Map (гарантирует уникальность):**
```typescript
const deptMap = new Map<string, number>()

positions.forEach(p => {
  const unit = normalizeUnit(p.department ?? '')
  if (!unit) return
  deptMap.set(unit, (deptMap.get(unit) ?? 0) + 1)
})
```

**Шаг 3 — Сортировка: Руководство всегда первым, остальные по алфавиту:**
```typescript
const sortedDepts = [...deptMap.entries()].sort(([a], [b]) => {
  if (a === 'Руководство') return -1
  if (b === 'Руководство') return 1
  return a.localeCompare(b, 'ru')
})
```

**Шаг 4 — Фильтрация правой части по нормализованному значению:**
```typescript
const filtered = positions.filter(p =>
  normalizeUnit(p.department ?? '') === selectedDept
)
```

## ЗАДАЧА 2 — Sticky-заголовок: убрать наложение на первую строку

### Проблема
Заголовок таблицы (`ДОЛЖНОСТЬ / ПОДРАЗДЕЛЕНИЕ / РУКОВОДИТЕЛЬ / КАРТОЧКА`)
прилипает с `top: 64px` и перекрывает первую строку данных при скролле.

### Решение

Контейнер таблицы должен иметь свой внутренний скролл, а sticky
работать внутри этого контейнера — тогда `top: 0`:

```tsx
{/* Обёртка с фиксированной высотой и overflow */}
<div style={{ overflowY: 'auto', maxHeight: 'calc(100vh - 260px)' }}>
  <table>
    <thead style={{
      position: 'sticky',
      top: 0,
      zIndex: 20,
      backgroundColor: '#0a0a1a',  // var(--card-bg) или похожий тёмный
      borderBottom: '1px solid rgba(0,229,255,0.3)'
    }}>
      <tr>
        <th>ДОЛЖНОСТЬ</th>
        <th>ПОДРАЗДЕЛЕНИЕ</th>
        <th>РУКОВОДИТЕЛЬ</th>
        <th>КАРТОЧКА</th>
      </tr>
    </thead>
    <tbody>
      {/* строки */}
    </tbody>
  </table>
</div>
```

Ключевое: `position: sticky; top: 0` работает корректно **только внутри
скроллящегося контейнера** (`overflow: auto/scroll`).
Если sticky навешан на `thead` у таблицы без скроллящегося контейнера —
он прилипает к viewport и может перекрывать контент.

### Также для режима «По подразделениям»
Правая часть (список должностей) — тоже обернуть в `overflow-y: auto`
с `max-height: calc(100vh - 260px)` чтобы скролл был внутренний,
а левая панель оставалась видна.
