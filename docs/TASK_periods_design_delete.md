## Проблема 2 — Нет удаления периода

### Бэкенд `backend/app/api/periods.py`

Добавить эндпоинт:
```
DELETE /api/periods/{id}
```

**Логика удаления:**
- Разрешено удалять только периоды со статусом `draft`
- Если статус `active` / `review` / `closed` → 400 с сообщением:
  «Нельзя удалить активный или закрытый период. Сначала переведите в статус draft»
- При удалении периода → каскадно удалять связанные `kpi_submissions` со статусом `draft`
- Если есть submissions со статусом `submitted` / `approved` → запретить удаление:
  «Период содержит отправленные или утверждённые отчёты. Удаление невозможно»
- Только роль `admin`

**Ответ:** `{"message": "Период удалён", "deleted_submissions": N}`

### Фронтенд `frontend/src/app/admin/periods/page.tsx`

- Кнопка «🗑 Удалить» на каждой карточке периода
  - Показывать только для статуса `draft`
  - Стиль: маленькая, `.cyber-btn` danger (красный), outline вариант
- При клике → модальное окно подтверждения:
  ```
  «Удалить период "Апрель 2026"?
  Это действие также удалит все черновики отчётов за этот период.
  Отменить | Удалить»
  ```
- Модальное окно в Dark Cyber стиле (тёмный фон, border accent)
- После успешного удаления → список обновляется без перезагрузки
