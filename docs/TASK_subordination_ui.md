# Задача: Просмотр подчинённости в admin-панели
> Для Claude Code | Прочитай `docs/BL_subordination.md` перед началом

## Цель
Добавить таб «Подчинённость» в admin-панель — только просмотр, без редактирования.
Редактирование будет добавлено после анализа Redmine API.

## Бэкенд

**Файл:** `backend/app/api/admin.py`

Добавить эндпоинт:

```
GET /api/admin/subordination
```

**Логика:**
1. Читать все должности из `KpiMappingService` (все 91 запись из KPI_Mapping.xlsx)
2. Для каждой должности читать `evaluator_role_id` из `SubordinationService`
3. Искать сотрудника по `evaluator_role_id` в таблице `employees` → получить имя
4. Искать сотрудника по `role_id` в таблице `employees` → получить имя занимающего должность

**Возвращать список объектов:**
```json
[
  {
    "role_id": "ЦТР_КОН_075",
    "pos_id": 75,
    "role_name": "Консультант",
    "unit": "VIII Уп. цифровой трансформации...",
    "employee_name": "Иванов Иван",
    "employee_redmine_id": 123,
    "evaluator_role_id": "ЦТР_ЗАМ_НАЧ_ОТД_073",
    "evaluator_name": "Бельтюгов Руслан",
    "status": "ok"
  }
]
```

**Статусы:**
- `ok` — evaluator_role_id задан И сотрудник с таким role_id есть в employees
- `no_evaluator` — evaluator_role_id не задан (null)
- `evaluator_missing` — evaluator_role_id задан, но сотрудника с таким role_id нет в employees
- `position_vacant` — должность есть в xlsx, но сотрудника с таким role_id нет в employees

**Сортировка:** по `pos_id` (от 1 до 91)

## Фронтенд

**Файл:** `frontend/src/app/admin/page.tsx`

Добавить новый таб **«Подчинённость»** (после существующих табов).

**Компонент таба:**

1. **Фильтр сверху** — dropdown «Все подразделения» + список уникальных `unit` из ответа API. Client-side фильтрация.

2. **Статистика** — строка из 4 бейджей:
   - `✅ Настроено: N`
   - `⚠️ Нет руководителя: N`
   - `❌ Руководитель не найден: N`
   - `👤 Вакансия: N`

3. **Таблица:**

| Должность | Сотрудник | Подразделение | Руководитель | Статус | |
|---|---|---|---|---|---|
| Консультант | Иванов И. | VIII Уп. ЦТР | Бельтюгов Р. | ✅ | Сообщить об ошибке |

   - «Должность» — сокращённо (без длинной строки unit)
   - «Сотрудник» — ФИО или «— вакансия —» серым
   - «Руководитель» — ФИО или «— не задан —» красным
   - «Статус» — цветной бейдж
   - Кнопка «Сообщить об ошибке» — пока `alert('Функция в разработке')`

4. **Дизайн** — Dark Cyber стиль (`.cyber-card`, таблица как в `/review`)

- Не трогать существующие табы admin-панели
