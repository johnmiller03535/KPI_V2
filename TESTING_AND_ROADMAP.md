## БИЗНЕС-ПРОЦЕСС СИСТЕМЫ

### Участники
- **Администратор** — настраивает систему, создаёт периоды
- **Сотрудник** — заполняет KPI-отчёт за период
- **Руководитель** — проверяет и утверждает отчёты подчинённых
- **Финансовый блок** — просматривает утверждённые отчёты

### Полный цикл отчётного периода

```
[АДМИНИСТРАТОР]
    │
    ▼
1. Проверить структуру (Redmine → Импорт из People → проверить ⚠️ без карточек)
    │
    ▼
2. Убедиться что у всех должностей есть KPI-карточки
    │
    ▼
3. Создание отчётного периода → задачи в Redmine
    │
    ▼ (автоматически создаются KPI-submission для каждого)

[СОТРУДНИК]
    │
    ▼
4. Списание трудозатрат в Redmine в течение периода
    │
    ▼
5. Открытие KPI-формы → «Загрузить саммари» → AI оценивает binary_auto
    │
    ▼
6. Отправить на проверку

[РУКОВОДИТЕЛЬ]
    │
    ▼
7. Получение уведомления в Telegram
    │
    ▼
8. binary_manual ✅/❌ → утверждение/отклонение

[ФИНАНСОВЫЙ БЛОК]
    │
    ▼
9. Просмотр утверждённых отчётов + PDF
```

Ключевые сценарии:
- [ ] Дерево подчинённости отображает всю иерархию корректно
- [ ] Бейджи карточек правильные (✅ / ⚠️)
- [ ] Клик ⚠️ в дереве → переход в KPI-карточки с нужной должностью
- [ ] Сайдбар карточек: все подразделения есть, счётчики верные
- [ ] Wizard: создать пустую карточку → добавить показатель → сохранить
- [ ] Wizard: скопировать карточку → проверить что показатели скопированы
- [ ] «+ Добавить показатель»: создать binary_auto, threshold, проверить форму
- [ ] Баннер после «Импорт из People»
