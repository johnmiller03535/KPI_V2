## ПЛАН ТЕСТИРОВАНИЯ ШАГА K

После реализации числовых KPI в форме сотрудника:

- [ ] threshold: поля числитель/знаменатель видны для threshold-показателей
- [ ] threshold: ввод данных → % считается правильно
- [ ] threshold: пороги применяются → правильный балл
- [ ] quarterly_threshold: определяется правильный квартал по дате периода
- [ ] quarterly_threshold: применяются пороги текущего квартала
- [ ] multi_threshold: отдельные поля для каждого подпоказателя
- [ ] multi_threshold: все выполнены → 100, хотя бы один нет → 0
- [ ] multi_binary: руководитель видит каждый подпоказатель отдельно
- [ ] Нарастающим итогом: данные суммируются с начала года
