## БИЗНЕС-ПРОЦЕСС СИСТЕМЫ

### Участники
- **Администратор** — настраивает систему, создаёт периоды
- **Сотрудник** — заполняет KPI-отчёт за период
- **Руководитель** — проверяет и утверждает отчёты подчинённых
- **Финансовый блок** — просматривает утверждённые отчёты

### Полный цикл отчётного периода

```
[АДМИНИСТРАТОР]
    │
    ▼
1. Настройка подчинённости (кто кому подчиняется)
    │
    ▼
2. Создание отчётного периода (месяц/квартал/год)
    │
    ▼
3. Создание задач в Redmine для всех сотрудников
    │
    ▼ (автоматически создаются KPI-submission для каждого)

[СОТРУДНИК]
    │
    ▼
4. Списание трудозатрат в Redmine в течение периода
   (по видам деятельности, привязанным к KPI)
    │
    ▼
5. Открытие KPI-формы → нажатие "Сгенерировать AI-оценку"
   - AI анализирует трудозатраты
   - binary_auto KPI → AI ставит оценку + summary
   - numeric KPI → сотрудник вводит факт вручную
   - binary_manual KPI → ожидают оценки руководителя
    │
    ▼
6. Просмотр результатов → отправка на проверку

[РУКОВОДИТЕЛЬ]
    │
    ▼
7. Получение уведомления в Telegram
    │
    ▼
8. Открытие формы проверки:
   - Просмотр AI-оценок (binary_auto)
   - Просмотр числовых показателей (numeric)
   - Ручная оценка показателей (binary_manual): ✅/❌
    │
    ▼
9. Утверждение или отклонение отчёта
   - При отклонении → сотрудник получает уведомление + причину
   - При утверждении → генерируется PDF, запись в Redmine

[ФИНАНСОВЫЙ БЛОК]
    │
    ▼
10. Просмотр утверждённых отчётов + PDF
    Сводка по подразделениям
```

## АРХИТЕКТУРНЫЕ ДОРАБОТКИ (запланированы)

### A. Управление подчинённостью через UI
**Проблема:** сейчас subordination хранится в JSON-файле, нельзя менять через интерфейс.
**Решение:**
- Таблица `subordination` в PostgreSQL
- Admin-панель → таб "Подчинённость"
- Дерево иерархии + редактирование
- Настройка ОБЯЗАТЕЛЬНА перед созданием периода
**Статус:** ❌ Не реализовано | Приоритет: 🔴 Высокий

### B. Activity Types → KPI маппинг
**Проблема:** AI угадывает к какому KPI относится задача.
**Решение:**
- В Redmine: виды деятельности настраиваются per-project
- В KPI_Mapping.xlsx: добавить колонку `activity_names`
- KpiEngineService: фильтровать time_entries по activity.name
**Статус:** ❌ Не реализовано | Приоритет: 🟡 Средний
