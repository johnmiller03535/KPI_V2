## БИЗНЕС-ПРОЦЕСС СИСТЕМЫ

### Участники
- **Администратор** — настраивает систему, создаёт периоды
- **Начальник управления / Зам. директора** — создаёт новые KPI-показатели (draft)
- **HR** — проверяет новые показатели (review) ← роль не реализована
- **Сотрудник** — заполняет KPI-отчёт за период
- **Руководитель** — проверяет и утверждает отчёты подчинённых
- **Финансовый блок** — просматривает утверждённые отчёты

### Полный цикл отчётного периода

```
[АДМИНИСТРАТОР]
    │
    ▼

    │
    ▼
2. Убедиться что у всех должностей есть KPI-карточки (все 91 ✅)

    ▼

    ▼

    │
    ▼

   Числовые показатели → сотрудник вводит числитель/знаменатель  ← шаг K
    │
    ▼
6. Отправить на проверку

[РУКОВОДИТЕЛЬ]
    │
    ▼
7. Уведомление в Telegram
    │
    ▼
8. multi_binary и binary_manual → ✅/❌ → утверждение/отклонение

[ФИНАНСОВЫЙ БЛОК]
    │
    ▼
9. Просмотр утверждённых отчётов + PDF
```

## ПЛАН ТЕСТИРОВАНИЯ ШАГА K

После реализации числовых KPI:

- [ ] threshold: поля числитель/знаменатель видны для threshold-показателей
- [ ] threshold: ввод данных → % считается правильно (числитель/знаменатель×100)
- [ ] threshold: пороги применяются сверху вниз → правильный балл
- [ ] absolute_threshold: одно поле ввода, прямое сравнение с порогами
- [ ] quarterly_threshold: определяется правильный квартал по дате окончания периода
- [ ] quarterly_threshold: Q1 (янв-март), Q2 (апр-июн), Q3 (июл-сен), Q4 (окт-дек)
- [ ] multi_threshold: отдельные поля для каждого подпоказателя
- [ ] multi_threshold: все выполнены → 100, хотя бы один нет → 0
- [ ] multi_binary: руководитель видит каждый подпоказатель отдельно
- [ ] multi_binary: все ✅ → 100, хотя бы одна ❌ → 0
- [ ] Нарастающим итогом: подсказка для сотрудника
